<p class="wp-block-paragraph">Strong security and governance are essential for any external-facing web presence. Power Pages now strengthens this with enhanced SharePoint permission management shifting from broad site collection access to precise, site-level controls.</p>



<h3 class="wp-block-heading" id="key-improvements">Key improvements</h3>



<ul class="wp-block-list">
<li class="wp-block-list-item">Site-level scoping instead of site collection wide access</li>



<li class="wp-block-list-item">Reduced exposure of unrelated SharePoint content</li>



<li class="wp-block-list-item">Stronger alignment with least-privilege security principles</li>
</ul>



<p class="wp-block-paragraph">This change ensures that Power Pages site only has access to the SharePoint resources it truly needs nothing more.</p>



<figure class="wp-block-image size-full"><img alt="Animated Gif Image" class="wp-image-134105" height="720" src="https://www.microsoft.com/en-us/power-platform/blog/wp-content/uploads/2026/04/Site-scoped-permission-slow.gif" width="1280" /></figure>



<h3 class="wp-block-heading" id="no-change-to-admin-experience-or-configuration-flow">No change to Admin experience or configuration flow</h3>



<p class="wp-block-paragraph">The existing setup experience for SharePoint document management remains consistent, including:</p>



<ul class="wp-block-list">
<li class="wp-block-list-item">Enabling document management for the environment</li>



<li class="wp-block-list-item">Configuring SharePoint integration in the Power Platform admin centre</li>



<li class="wp-block-list-item">Setting up forms and table permissions to control user access</li>
</ul>



<p class="wp-block-paragraph">These steps are described in detail in the documentation:<br />👉 <a href="https://learn.microsoft.com/en-us/power-pages/configure/manage-sharepoint-documents" rel="noreferrer noopener" target="_blank">Manage SharePoint documents</a> </p>



<p class="wp-block-paragraph">The improvement is focused on how permissions are granted behind the scenes, not on introducing additional complexity for makers or admins.</p>
